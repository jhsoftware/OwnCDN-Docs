# UI / Service Status - Storage

[#INCLUDE](parts/ss_id_type.md)

## Last sync time

(Only available when the "Replica" option is enabled for the service)

The last time the database was synchronized with the Replication Source server.

## Last sync result

(Only available when the "Replica" option is enabled for the service)

The result of the last synchronization with the Replication Source server.


## Current files

The number of files currently stored in the service.


## Old versions

The number of old versions of files currently stored in the service.

A "Purge..." button will be available if there are any old versions. This allows you to purge / delete the old versions back to a point-in-time.


## Deletions

The number of files marked for deletion currently stored in the service.
